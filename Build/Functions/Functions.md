# Game function roadmap

- Phase 4 introduces `runCommand` as the first secure backend callable.
- Runtime command rewards and cooldown state are validated by backend logic.
- Phase 5 adds command lesson purchasing / unlock flow in Black Market and Index tracking.
- Phase 6 adds casino wager logic, streak tracking, and Flux badge rewards.
- Phase 7 adds live stock tick simulation and buy/sell share management APIs.
- Phase 8 adds PvP queue/ready/match/result state transitions and payout handling.
- Future phases move all economy and combat outcomes to full server-authoritative Cloud Functions.
